\section{Producto operativo: fichas, mapas y criterios de documentación}

Como ocurre en varias ICR orientadas por la conservación del patrimonio construido, esta investigación no concluye únicamente con una interpretación teórica. El producto operativo será un conjunto organizado de fichas descriptivas, mapas generalizados, síntesis comparativa y criterios básicos de documentación y conservación preventiva sobre construcciones tradicionales hñähñu.

Este producto mantiene el alcance de la investigación dentro de lo que sí está documentado: cuatro construcciones, fotografías, relatos y ubicación territorial. Su función no es proponer una restauración ni cerrar una clasificación definitiva, sino ordenar la evidencia disponible para que pueda discutirse, revisarse y ampliarse en una etapa posterior.

\subsection{Objetivo del producto operativo}

El producto tendrá tres funciones principales:

\begin{enumerate}
	\item \textbf{Documentar}: reunir información clara sobre cada construcción registrada, con fotografías, descripción del sistema y ubicación generalizada.
	\item \textbf{Comparar}: permitir la lectura entre sistemas constructivos, materiales, usos y estados de conservación.
	\item \textbf{Orientar}: proponer criterios mínimos para continuar el registro sin perder la relación entre construcción, territorio y relato local.
\end{enumerate}

De esta manera, el producto funcionará como una herramienta de documentación patrimonial y como base para trabajo posterior, no como una propuesta de intervención constructiva.

\subsection{Componentes del producto}

El producto operativo se organiza en cuatro componentes:

\begin{enumerate}
	\item \textbf{Fichas descriptivas}: una ficha por construcción, con clave anónima, sistema constructivo, materiales, uso, estado, cambios y relato asociado.
	\item \textbf{Mapas generalizados}: mapas de contexto regional, comunitario y local, evitando publicar coordenadas exactas.
	\item \textbf{Síntesis comparativa}: cuadro de comparación entre los sistemas identificados, sus materiales principales y sus transformaciones visibles.
	\item \textbf{Criterios de documentación}: recomendaciones básicas para registrar nuevas construcciones tradicionales con la misma lógica de campo.
\end{enumerate}

Estos componentes permiten que la información sea legible dentro de la ICR y, al mismo tiempo, útil para una eventual devolución comunitaria o continuidad académica.

\subsection{Criterios de documentación y conservación preventiva}

A partir del diagnóstico se formularán criterios básicos para documentar y conservar estas construcciones sin forzar una propuesta de restauración. Los criterios preliminares son:

\begin{enumerate}
	\item \textbf{Registrar antes de intervenir}: documentar fotografía, ubicación general, uso y relato antes de cualquier modificación.
	\item \textbf{Reconocer el sistema completo}: no separar muros, cubierta, piso, patio, vegetación y usos cotidianos como si fueran elementos aislados.
	\item \textbf{Priorizar compatibilidad material}: evitar sustituciones que generen incompatibilidades físicas o alteren de forma irreversible la lectura constructiva.
	\item \textbf{Registrar cambios como parte de la historia}: distinguir entre deterioro, mantenimiento, adaptación y sustitución.
	\item \textbf{Proteger la información sensible}: generalizar ubicaciones y anonimizar construcciones cuando la publicación pueda afectar a propietarios o habitantes.
	\item \textbf{Incluir la voz local}: asociar cada ficha con relatos de uso, memoria o mantenimiento para no reducir la construcción a su materialidad.
\end{enumerate}

Estos criterios serán la base ética y metodológica del producto operativo. Su función es evitar que las construcciones se presenten como objetos aislados y mantener visible la relación entre materialidad, territorio y saber local.

\subsection{Privacidad y publicación}

La versión académica del producto no deberá publicar coordenadas exactas ni datos personales de propietarios o entrevistados. Cuando una fotografía permita identificar a una persona, un predio o una situación sensible, se usará solo con autorización explícita o se reservará para consulta interna.

El trabajo podrá manejar dos niveles de información: una versión de análisis con datos completos para revisión académica y una versión de presentación con ubicación generalizada. Esta distinción permite aprovechar la precisión del trabajo cartográfico sin exponer indebidamente a las comunidades.
